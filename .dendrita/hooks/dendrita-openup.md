## What is this Hook?

This hook documents the expected behavior that Cursor must apply when the user requests that dendrita "open up" and reveal hidden information, patterns, connections, or non-obvious insights.

**Purpose:** Reveal hidden information, patterns, connections, and insights that are not obvious at first glance, helping the user discover relationships, opportunities, and areas for improvement in their dendrita system.

**Difference with other hooks:**
- This hook focuses on revealing hidden or non-obvious information
- Generates insights and connections that are not evident
- Analyzes patterns and relationships between components
- Provides new perspectives on the system

### 2. Revelation Process

**CRITICAL:** When the hook is activated, Cursor MUST execute ALL of the following steps in order. DO NOT skip any step:

**Step 0: Acknowledge Activation**
- Immediately acknowledge that you're executing the dendrita-openup hook
- Confirm that you understand the request
- Begin the revelation process

When the hook is activated, Cursor must:

#### Step 1: Pattern Analysis (Separate Dev and Work)

**CRITICAL:** Analyze patterns separately for DEV (infrastructure) and WORK (projects):

**DEV (Infrastructure) Analysis:**
```markdown
1. Analyze hooks, skills, agents, and scripts:
   - Hooks not used recently or with issues
   - Skills not used recently
   - Agents without activation in the last period
   - Scripts not executed recently
   - Scripts with errors or problems
   - Automation opportunities
   - Duplicate or redundant scripts
   - Infrastructure components that need attention

2. Analyze infrastructure patterns:
   - Components with old pending updates
   - Infrastructure gaps or missing pieces
   - Optimization opportunities in infrastructure
   - Technical debt or maintenance needs
```

**WORK (Projects) Analysis:**
```markdown
1. Analyze active projects to identify:
   - Projects without recent updates
   - Projects with old pending tasks
   - Projects with stakeholders without recent interaction
   - Projects with objectives without progress

2. Analyze work patterns:
   - Workspaces with low activity
   - Projects with unexploited potential
   - Work dependencies or blockers
   - Collaboration opportunities
```

#### Step 2: Connection Identification (Separate Dev and Work)

**CRITICAL:** Identify connections separately for DEV and WORK:

**DEV (Infrastructure) Connections:**
```markdown
1. Identify connections between infrastructure components:
   - Skills that could be applied to specific projects
   - Agents that could help in current projects
   - Scripts that could automate manual tasks
   - Hooks that could improve workflows
   - Infrastructure dependencies or relationships

2. Identify infrastructure-to-work connections:
   - Skills/agents that could benefit work projects
   - Scripts that could automate work tasks
   - Infrastructure improvements that would help work
```

**WORK (Projects) Connections:**
```markdown
1. Identify connections between projects:
   - Projects with shared stakeholders
   - Projects with related objectives
   - Projects with non-explicit dependencies
   - Synergy opportunities between projects

2. Identify connections between workspaces:
   - Workspaces with related projects
   - Workspaces with shared stakeholders
   - Collaboration opportunities between workspaces
   - Work patterns between workspaces
```

#### Step 3: Insight Generation (Separate Dev and Work)

**CRITICAL:** Generate insights separately for DEV and WORK:

**DEV (Infrastructure) Insights:**
```markdown

   - Infrastructure gaps or missing pieces

2. Generate insights about infrastructure components:
   - Components that need attention
   - Components with unexploited potential
   - Infrastructure dependencies or issues
   - Consolidation opportunities in infrastructure
```

**WORK (Projects) Insights:**
```markdown
1. Generate insights about projects:
   - Projects that need attention

2. Generate insights about stakeholders:
```
