# Dendrita Work Status Hook

Behavior reference for Cursor - automatic generation of consolidated work status reports from all project contexts.

---

## What is this Hook?

This hook documents the expected behavior that Cursor must apply when the user requests to see all pending tasks and status of all projects and work.

**Purpose:** Generate a consolidated report of all pending tasks, in-progress tasks, blocked tasks, and project statuses across all workspaces by reading `project_context.json` files automatically.

**Key advantage:** Uses existing `project_context.json` files (generated by `update-project-context.ts`) to automatically consolidate information without manual data entry.

### 2. Work Status Report Process

When dendrita-work-status behavior is activated, Cursor must:

#### Step 1: Generate Report

1. **Recommended: Execute the master sync script (updates everything):**
   ```bash
   tsx .dendrita/integrations/scripts/pipelines/context-pipeline/sync-all-context.ts
   ```
   This will update project contexts, user/workspace contexts, and generate the work status report in the correct order.

2. **Alternative: Execute only the report generation script:**
   ```bash
   tsx .dendrita/integrations/scripts/pipelines/context-pipeline/generate-work-status-report.ts
   ```
   Use this if contexts are already up to date.

3. **The script will:**
   - Find all active projects across all workspaces
   - Read all `project_context.json` files
   - Consolidate all pending, in-progress, and blocked tasks
   - Generate report in `.dendrita/dashboards/work-status-report.md` and `.json`

4. **If workspace filter is needed:**
   ```bash
   tsx .dendrita/integrations/scripts/pipelines/context-pipeline/sync-all-context.ts --workspace [workspace]
   ```
   Or for report only:
   ```bash
   tsx .dendrita/integrations/scripts/pipelines/context-pipeline/generate-work-status-report.ts --workspace [workspace]
   ```

#### Step 2: Read and Present Report

1. **Read the generated report:**
   - Location: `.dendrita/dashboards/work-status-report.md`
   - If report doesn't exist or is outdated, generate it first

2. **Present the information:**
   - Show summary (total projects, tasks by status)
   - Show projects organized by workspace
   - Show all pending tasks grouped by workspace and project
   - Show all in-progress tasks
   - Show all blocked tasks

3. **If user asks for specific information:**
   - Filter by workspace if mentioned
   - Filter by project if mentioned
   - Filter by task status if mentioned
   - Filter by priority if mentioned

#### Step 3: Update Context Files (if needed)

1. **Check if project_context.json files are up to date:**
   - If user is actively working and needs accurate data â†’ Automatically update contexts first
   - **Recommended:** Run master sync script: `tsx .dendrita/integrations/scripts/pipelines/context-pipeline/sync-all-context.ts`
   - **Alternative:** Run individual scripts:
     - `tsx .dendrita/integrations/scripts/pipelines/context-pipeline/update-project-context.ts`
     - `tsx .dendrita/integrations/scripts/pipelines/context-pipeline/update-context.ts`
     - `tsx .dendrita/integrations/scripts/pipelines/context-pipeline/generate-work-status-report.ts`

2. **Then generate the report (if not using master sync):**
   - Run the work status report script
   - Present the results

**Note:** The master sync script (`sync-all-context.ts`) executes all steps in the correct order automatically.

### 3. Report Structure

The generated report includes:

1. **Summary:**
   - Total projects (active, paused, completed)
   - Total tasks by status (pending, in-progress, blocked, completed)

2. **By Workspace:**
   - Projects organized by workspace
   - Status of each project
   - Last activity date
   - Task counts per project
   - Top pending tasks per project

3. **All Pending Tasks:**
   - Complete list of all pending tasks
   - Organized by workspace and project
   - Includes priority and due dates if available

4. **All In Progress Tasks:**
   - Complete list of all in-progress tasks
   - Organized by workspace and project

5. **All Blocked Tasks:**
   - Complete list of all blocked tasks
   - Organized by workspace and project

### 4. Output Format

**For Cursor:**

When presenting the report to the user:

1. **Show summary first:**
   ```markdown
   ## Work Status Summary
   - Total Projects: X (Y active, Z paused)
   - Pending Tasks: X
   - In Progress: Y
   - Blocked: Z
   ```

2. **Show by workspace:**
   ```markdown
   ### [Workspace] (X projects)
   - [Project] [status] - Last Activity: [date]
     - Tasks: X pending, Y in progress, Z blocked
   ```

3. **Show detailed task lists if requested:**
   - All pending tasks
   - All in-progress tasks
   - All blocked tasks

### Script Location

- **Script:** `.dendrita/integrations/scripts/pipelines/context-pipeline/generate-work-status-report.ts`
- **Output:** `.dendrita/dashboards/work-status-report.md`
- **JSON Output (optional):** `.dendrita/dashboards/work-status-report.json`

### Dependencies

- Requires `project_context.json` files to exist (generated by `update-project-context.ts`)
- Reads from `workspaces/[workspace]/ðŸš€ active-projects/[project]/project_context.json`

### Automatic Updates

**When to update project contexts:**

- Before generating report if contexts are outdated
- If user is actively working and needs accurate data
- If last update was more than 24 hours ago

**When to regenerate report:**

- When user explicitly requests it
- If report is older than project contexts
- If user mentions "update" or "refresh" report
