# Example User Structure

This is an **example user structure** demonstrating how to set up a user in the dendrita system. This structure is public and can be used as a template for creating new users.

## Structure

```
.dendrita/users/example-user/
├── profile.example.json              # Example structure for user profile
├── context.json.example              # Example structure for user context JSON
├── workspace-defaults.example.json   # Example structure for workspace defaults
├── scrapers-config.json.example      # Example structure for scrapers configuration
├── reddit-comments-monitoring.json.example  # Example structure for Reddit monitoring
├── profiles/                         # Additional profiles templates
│   └── profile-workspace.example.json  # Example structure for workspace profile
├── agents/                           # Specialized agents (domain knowledge)
│   ├── example-agent.md              # Example agent template
│   └── README.md                     # Agent documentation (optional)
├── skills/                           # Contextual knowledge skills (domain knowledge)
│   ├── skill-rules.json              # Skill activation rules
│   ├── example-skill/                # Example skill
│   │   └── SKILL.md                  # Skill documentation
│   └── README.md                     # Skills documentation (optional)
├── work-modes/                       # General work preferences
│   ├── user-work-mode.md             # General work preferences
│   └── README.md                     # Work-modes documentation (optional)
└── README.md                         # This file
```

## Components

### profile.example.json
Example structure for user profile:
- **Location:** `.dendrita/users/[user-id]/profile.json`
- **Structure:**
  - Basic information (name, email, workspace)
  - Work preferences (language, communication style)
  - Professional context (experience, expertise, skills)
  - Dendrita settings (alias, name, auto-activation)
  - Integration settings
- **See:** `profile.example.json` for complete structure with comments

### context.json.example
Example structure for user context JSON:
- **Generated by:** `update-context.ts`
- **Location:** `.dendrita/users/[user-id]/context.json`
- **Structure:**
  - `quickReference` - Fast lookup with recent memories, active workspaces, recent files
  - `memories` - All memories from all projects and workspaces (propagated from projects)
  - `summary` - Statistics aggregated from all memories
- **See:** `context.json.example` for complete structure with comments

### Configuration Templates

#### workspace-defaults.example.json
Example structure for workspace defaults configuration:
- **Location:** `.dendrita/users/[user-id]/workspace-defaults.json`
- **Purpose:** Configure which profile to use by default in each workspace
- **Structure:**
  - `workspace_profiles` - Mapping of workspace names to profile IDs
  - `default_profile` - Fallback profile when workspace is not configured
- **See:** `workspace-defaults.example.json` for complete structure with comments

#### profiles/profile-workspace.example.json
Example structure for workspace-specific profile:
- **Location:** `.dendrita/users/[user-id]/profiles/workspace-[workspace-name].json`
- **Purpose:** Profile customized for a specific workspace
- **Structure:**
  - Workspace-specific preferences
  - Workspace-specific work context
  - Workspace-specific settings
- **See:** `profiles/profile-workspace.example.json` for complete structure with comments

#### scrapers-config.json.example
Example structure for scrapers configuration:
- **Location:** `.dendrita/users/[user-id]/scrapers-config.json`
- **Purpose:** Configure scraping settings for Google Calendar, Gmail, etc.
- **Structure:**
  - Calendar scraping settings
  - Gmail scraping settings
  - Drive scraping settings
- **See:** `scrapers-config.json.example` for complete structure with comments

#### reddit-comments-monitoring.json.example
Example structure for Reddit comments monitoring:
- **Location:** `.dendrita/users/[user-id]/reddit-comments-monitoring.json`
- **Purpose:** Configure monitoring of Reddit posts and comments
- **Structure:**
  - List of posts to monitor
  - Workspace/project context for each post
  - Monitoring settings
- **See:** `reddit-comments-monitoring.json.example` for complete structure with comments

### agents/
Specialized agents for complex multi-step tasks:
- Each agent is a standalone `.md` file
- Contains instructions, principles, and expected outputs
- Domain-specific knowledge (user-specific)
- **Location:** `.dendrita/users/[user-id]/agents/`

### skills/
Contextual knowledge and inline guidance:
- Each skill has its own folder with `SKILL.md`
- Contains methodology, patterns, and best practices
- Domain-specific knowledge (user-specific)
- Activated based on `skill-rules.json`
- **Location:** `.dendrita/users/[user-id]/skills/`

### work-modes/
General work preferences:
- `user-work-mode.md` defines general work preferences
- Applied to all sessions as base behavior
- User-specific preferences
- **Location:** `.dendrita/users/[user-id]/work-modes/`

## Paradigm

**Agents and skills are user-specific domain knowledge**, not generic technical infrastructure. They contain:
- User-specific methodologies
- Domain knowledge (sustainability, social impact, etc.)
- Personal work patterns and preferences

They are stored in `.dendrita/users/[user-id]/` to reflect that they contain private, domain-specific knowledge rather than generic infrastructure.

## How to Use This Template

1. **Copy the structure:**
   ```bash
   cp -r .dendrita/users/example-user .dendrita/users/[your-user-id]
   ```

2. **Create your profile:**
   - Copy `profile.example.json` to `profile.json`
   - Replace example information with your own
   - Configure your preferences and settings

3. **Set up configuration files:**
   - Copy `workspace-defaults.example.json` to `workspace-defaults.json` and configure workspace profiles
   - Copy `scrapers-config.json.example` to `scrapers-config.json` if using scrapers
   - Copy `reddit-comments-monitoring.json.example` to `reddit-comments-monitoring.json` if monitoring Reddit
   - Copy `profiles/profile-workspace.example.json` to `profiles/workspace-[workspace-name].json` for each workspace profile

4. **Customize agents:**
   - Replace `example-agent.md` with your own agents
   - Each agent should be domain-specific to your work

5. **Create your skills:**
   - Replace `example-skill` with your own skills
   - Update `skill-rules.json` with activation rules

6. **Customize work-mode:**
   - Edit `user-work-mode.md` with your personal preferences
   - Define how you prefer to work and communicate

## Integration with dendrita

This user structure integrates with:
- `.cursorrules` - Configuration for Cursor
- `.dendrita/settings.json` - System metadata
- `.dendrita/hooks/` - Behavior references

## Notes

- Replace all example content with your own domain knowledge
- Maintain consistency with dendrita system structure
- Keep domain knowledge separate from generic infrastructure
