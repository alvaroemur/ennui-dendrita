# Project Files Templates

This directory contains templates for the three standard project files that are automatically parsed by the `update-project-context.ts` script.

## Available Templates

### Markdown Templates (Source Files)

#### current-context-template.md
**Purpose:** Template for current project context (renamed to current_context.md)  
**Parsed by:** `parseCurrentContext()` in `.dendrita/integrations/scripts/pipelines/context-pipeline/update-project-context.ts`

**Required sections:**
- `## Estado Actual` / `## Current Status` / `## Status` - Current status
- `## SESSION PROGRESS` / `## Progreso` / `## Session Progress` - Session progress (with `### YYYY-MM-DD` for dates)
- `## Decisiones` / `## Decisions` / `## Recent Decisions` - Recent decisions (list with `-` or `*`)
- `## Blockers` / `## Bloqueadores` / `## ObstÃ¡culos` - Blockers (list with `-` or `*`)
- `## PrÃ³ximos Pasos` / `## Next Steps` - Next steps (list with `-` or `*`)

**See:** `current-context-template.md` for detailed structure and parsing requirements

#### tasks-template.md
**Purpose:** Template for project tasks  
**Parsed by:** `parseTasks()` in `.dendrita/integrations/scripts/pipelines/context-pipeline/update-project-context.ts`

**Required format:**
- Checkbox format: `- [ ]` (pending), `- [x]` (completed), `- [~]` (blocked)
- Also accepts: `* [ ]` instead of `- [ ]`
- Optional metadata in description:
  - `[high]`, `[medium]`, `[low]`, `[urgent]` for priority
  - `@usuario` for assignee
  - `due: YYYY-MM-DD` for due date

**See:** `tasks-template.md` for detailed structure and parsing requirements

   ```bash

   ```

2. **Customize the templates:**
   - Remove the frontmatter and notes sections if desired
   - Fill in your project-specific content
   - Maintain the required structure for parsing

3. **Update project_context.json:**
   ```bash
   tsx .dendrita/integrations/scripts/pipelines/context-pipeline/update-project-context.ts --workspace [workspace] --project [project]
   ```

## Important Notes

### Structure Requirements
- **All templates include explicit references** to the parsing requirements
- **Comments in templates** explain what the parser looks for
- **Variants accepted** are documented in each template
- **Examples** are provided for clarity

### Parsing Behavior
- If a section is not found, the parser returns empty structure but keeps `rawContent`
- The parser is flexible and accepts Spanish/English variants
- All content is preserved in `rawContent` of the JSON
- Structured sections are extracted into specific fields

### Template Maintenance
- **Keep the reference comments** in your working copies for easy review
- **Update templates** if the parser changes
- **Document any custom sections** you add
