# Workspace Template

This is a **workspace template** for creating new workspaces within dendrita.

**Location:** `.dendrita/templates/workspace-template/`  
**Purpose:** Reference template for creating new workspaces

## Structure

Each workspace must contain the following folders:

- `active-projects/` - Active projects
- `_archived-projects/` - Completed projects
- `best-practices/` - Templates and company-specific methodologies
- `company-management/` - General company management (optional)
- `commercial-management/` - Commercial management and sales proposals (optional)
- `stakeholders/` - Allies and stakeholder management
- `tools-templates/` - Reusable tools and templates

**Note:** Specialized work-modes have been consolidated into agents and skills in `.dendrita/users/[user-id]/`. See `.dendrita/users/example-user/` for reference structure.

## Optional Files

- `config-estilo.json` - Company-specific style rules
- `scrapers-config.json.example` - Scraper configuration template
- `workspace-context.json.example` - Example structure for workspace context JSON

## Project Structure

Each project within `active-projects/` or `_archived-projects/` must follow the standard structure:

- `master_plan.md` - Project master plan
- `current_context.md` - Current state and decisions
- `tasks.md` - Task list with status
- `project_context.json` - Generated context JSON (auto-generated)
- `README.md` - Project documentation (optional)

**Templates available:** See `.dendrita/templates/workspace-template/project-files/` for templates with explicit parsing requirements.

## How to Use This Template

1. **Copy the template structure:**
   ```bash
   cp -r .dendrita/templates/workspace-template workspaces/[new-workspace-name]
   ```

2. **Customize the workspace:**
   - Replace "Template" references with your company name
   - Customize README.md files in each folder according to company needs
   - Add specific content in each folder as needed

3. **Set up your workspace:**
   - Create projects in `active-projects/`
   - Use templates from `project-files/` for new projects (see below)
   - Add methodologies in `best-practices/`
   - Configure products in `products/`
   - Set up stakeholder management in `stakeholders/`
   - Set up commercial management in `commercial-management/` (if applicable)

4. **Initialize new projects:**

## Project File Templates

**Location:** `.dendrita/templates/workspace-template/project-files/`

Templates for the three standard project files with explicit parsing requirements:

- `master-plan-template.md` - Template for project master plan (renamed to master_plan.md, parsed by `parseMasterPlan()`)
- `current-context-template.md` - Template for current context (renamed to current_context.md, parsed by `parseCurrentContext()`)
- `tasks-template.md` - Template for tasks (parsed by `parseTasks()`)
- `README.md` - Documentation on project file templates

**Important:** All templates include explicit references to parsing requirements, making it easy to review what structure is needed for automatic JSON generation.

**See:** `.dendrita/templates/workspace-template/project-files/README.md` for detailed documentation

## Workspace Context Template

**Location:** `.dendrita/templates/workspace-template/workspace-context.json.example`

Example structure for workspace context JSON:
- **Generated by:** `update-context.ts`
- **Location:** `workspaces/[workspace]/context.json`
- **Structure:**
  - `quickReference` - Fast lookup filtered by workspace
  - `memories` - Memories filtered from user context by workspace
  - `summary` - Statistics for this workspace only
- **See:** `workspace-context.json.example` for complete structure with comments
